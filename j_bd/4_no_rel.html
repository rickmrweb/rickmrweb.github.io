Diseño no Relacional

La base de datos es Firestore de Firebase. Las entidades se almacenan en colecciones, que consisten en una lista de identificadores que tienen asociados documentos. Cada documento contiene campos que tienen un nombre y llevan asociado un valor.

Los archivos se cargan en el servicio de Storage, donde se usará el id en la colección Usuario como nombre del archivo.

Para la asociaciones a 1 (1-1 o M-1) se usa el id del objeto relacionado. En este caso, el campo pasatiempoId de la colección Usuario hace referencia al id de la colección Pasatiempo.

El campo usuarioId de la colección Mensaje hace referencia al id de la colección Usuario.

Para la asociaciones a muchos (1-M o M-N) se usa un arreglo con los id de los objetos relacionados. En este caso, el campo rolIds de la colección Usuario hace referencia a algunos ids de la colección Rol.

El identificador de la colección Pasatiempo se genera automáticamente.

La colección Usuario se procesa más fácilmente si el id es el email. Un problema es que el usuario no puede cambiar de email.

Los datos de la colección Rol son usados directamente en el código. El desarrollador asigna los identificadores y llena los datos directamente en la base de datos desde el inicio.
